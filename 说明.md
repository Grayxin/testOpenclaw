# 打包
安装依赖: 运行 pnpm install 安装必要的依赖库。
执行构建: 运行 pnpm build 命令。
该命令会利用 tsdown 和内置脚本，将 src/ 下的核心代码和 extensions/ 下的插件全量编译并打包到根目录的 dist/ 文件夹中。
产出物: dist/ 文件夹就是你最终需要的纯净后端包，可以直接部署在服务器上运行。

# 初始化
node dist/index.js onboard

# 运行gateway

# 全局配置文件
src/config/paths.ts 

1. 变量含义解析
NEW_STATE_DIRNAME = ".openclaw"

含义：当前版本默认的状态目录名。
位置：通常在你的用户根目录下（例如 C:\Users\Gray\.openclaw）。
作用：这个文件夹是 OpenClaw 的“家”，里面存放着 openclaw.json、日志 (logs)、会话 (sessions)、缓存 (cache) 等所有持久化数据。
CONFIG_FILENAME = "openclaw.json"

含义：主配置文件名。
作用：程序启动时，会去 .openclaw 文件夹里找这个 JSON 文件。你之前修改的 API Token、模型列表等都在这个文件里。
LEGACY_STATE_DIRNAMES = [".clawdbot"] 和 LEGACY_CONFIG_FILENAMES = ["clawdbot.json"]

含义：旧版遗留的名称。
背景：OpenClaw 以前的名字叫 Clawdbot。
作用：为了兼容老用户，如果程序没找到 .openclaw 文件夹，它会尝试去找 .clawdbot。这属于“历史兼容代码”，如果你是新安装的，可以不用管它们。
2. 它们是全局的吗？
是的，它们在代码层面上是“路径定义的全局源头”。

整个项目（包括 Gateway、命令行工具、各种插件）在涉及“我该去哪读配置”时，都会调用这个 paths.ts 文件里的函数（如 resolveStateDir）。

如果你想彻底把项目改名（比如改成你的 Ailit）： 你不仅要改这几个常量，还要配合环境变量（比如 OPENCLAW_HOME）来使用。

总结
这几行代码回答了程序的一个根本问题：“我的配置文件叫什么？它藏在用户电脑的哪个文件夹里？”

目前你的情况：
配置文件夹：~/.openclaw
配置文件：~/.openclaw/openclaw.json
如果你想把项目改造成你自己的品牌，这里就是修改路径名称的最关键入口。
