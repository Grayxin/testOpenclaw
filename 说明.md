# 打包问题
如果你之后打包了项目，只使用 `dist` 目录下的文件，那么你的启动方式会从 `pnpm` 切换到直接使用 `node`。

### 1. 如何在 `dist` 中启动？

在没有 `pnpm` 的环境下（比如生产环境或打包后），你应该直接使用 `node` 运行入口文件：

*   **执行 Setup (初始化配置)**：
    ```powershell
    node dist/index.js onboard
    ```
    *(注意：`onboard` 是推荐的新手引导命令，`setup` 也是类似的。你刚才尝试 `gateway run` 报错就是因为还没执行这一步。)*

*   **正式运行 Gateway**：
    ```powershell
    node dist/index.js gateway run
    ```

### 2. 为什么刚才 `node dist/index.js gateway run` 报错了？

报错信息显示：
> `Missing config. Run openclaw setup or set gateway.mode=local (or pass --allow-unconfigured).`

这说明程序**启动成功了**（说明 `node dist/index.js` 确实是正确的入口），但因为你是第一次运行，本地还没有 `.openclaw/openclaw.json` 配置文件，所以它拒绝运行 Gateway。

**解决办法：**
先执行一次 `node dist/index.js onboard`，按照提示输入你的 API Key 和配置，生成配置文件后，再运行 `gateway run` 就可以了。

### 3. 注意事项（关于打包）

*   **依赖项**：虽然你只用 `dist` 里的 JS 文件，但这些 JS 文件仍然会去 `node_modules` 里找依赖（比如 `zod`、`json5` 等）。所以打包部署时，除了 `dist`，你通常还需要 `package.json` 并执行 `pnpm install --prod` 来获取运行时的依赖。
*   **入口文件**：在 OpenClaw 中，`dist/index.js` 是主入口，它负责解析你后面带的 `onboard`、`gateway run` 等所有指令。

**总结：**
你现在的直觉是完全正确的！只需要把 `pnpm openclaw` 替换成 `node dist/index.js` 即可。 建议你现在先在本地跑通 `node dist/index.js onboard`。



# 请求问题
从你的终端截图来看，你已经成功运行了 Gateway，虽然中间报了一个配置错误，但它最终显示 `ready` 了。

关于你如何通过 Postman 请求这个 Gateway，以及那个配置错误，我为你整理如下：

### 1. 如何通过 Postman 请求？

OpenClaw 的 Gateway 默认提供了一个 **OpenAI 兼容接口**，你可以像调用 GPT 一样调用它。

*   **URL**: `http://127.0.0.1:18789/v1/chat/completions`
*   **Method**: `POST`
*   **Headers**:
    *   `Authorization`: `Bearer e51c05864d8332a9c8b0c8c837550ecb53d4f513c239c989` (这是你配置里的 token)
    *   `Content-Type`: `application/json`
*   **Body (JSON)**:
    ```json
    {
      "model": "main",
      "messages": [
        { "role": "user", "content": "你好，请问你是谁？" }
      ]
    }
    ```
    *(注意：`model` 填 `main` 或者是你配置的具体 Agent ID。)*

---

### 2. 修复配置错误：`Unrecognized key: "sandbox"`

虽然 Gateway 启动了，但那个报错（`agents: Unrecognized key: "sandbox"`）说明你的 `openclaw.json` 格式有一点点小瑕疵。在当前的 OpenClaw 版本中，`sandbox` 应该在 `agents.defaults` 之外，或者它的位置不对。

**你可以尝试运行以下指令让系统自动修复它：**

```powershell
node dist/index.js doctor --fix
```

这个指令会自动扫描你的配置文件并把不符合规范的字段挪到正确的位置。
